          <section id="making-a-pypi-friendly-readme">
<h1>Making a PyPI-friendly README<a class="headerlink" href="#making-a-pypi-friendly-readme" title="Link to this heading">¶</a></h1>
<p>README files can help your users understand your project and can be used to set your project’s description on PyPI.
This guide helps you create a README in a PyPI-friendly format and include your README in your package so it appears on PyPI.</p>
<section id="creating-a-readme-file" class="">
<h2>Creating a README file<a class="headerlink" href="#creating-a-readme-file" title="Link to this heading">¶</a></h2>
<p>README files for Python projects are often named <code class="docutils literal notranslate"><span class="pre">README</span></code>, <code class="docutils literal notranslate"><span class="pre">README.txt</span></code>, <code class="docutils literal notranslate"><span class="pre">README.rst</span></code>, or <code class="docutils literal notranslate"><span class="pre">README.md</span></code>.</p>
<p>For your README to display properly on PyPI, choose a markup language supported by PyPI.
Formats supported by <a class="reference external" href="https://github.com/pypa/readme_renderer">PyPI’s README renderer</a> are:</p>
<ul class="simple">
<li><p>plain text</p></li>
<li><p><a class="reference external" href="https://docutils.sourceforge.io/rst.html">reStructuredText</a> (without Sphinx extensions)</p></li>
<li><p>Markdown (<a class="reference external" href="https://github.github.com/gfm/">GitHub Flavored Markdown</a> by default,
or <a class="reference external" href="https://commonmark.org/">CommonMark</a>)</p></li>
</ul>
<p>It’s customary to save your README file in the root of your project, in the same directory as your <code class="file docutils literal notranslate"><span class="pre">setup.py</span></code> file.</p>
</section>
<section id="including-your-readme-in-your-package-s-metadata" class="active">
<h2>Including your README in your package’s metadata<a class="headerlink" href="#including-your-readme-in-your-package-s-metadata" title="Link to this heading">¶</a></h2>
<p>To include your README’s contents as your package description,
set your project’s <code class="docutils literal notranslate"><span class="pre">Description</span></code> and <code class="docutils literal notranslate"><span class="pre">Description-Content-Type</span></code> metadata,
typically in your project’s <code class="file docutils literal notranslate"><span class="pre">setup.py</span></code> file.</p>
<div class="admonition seealso">
<p class="admonition-title">See also</p>
<ul class="simple">
<li><p><a class="reference internal" href="../../specifications/core-metadata/#description-optional"><span class="std std-ref">Description</span></a></p></li>
<li><p><a class="reference internal" href="../../specifications/core-metadata/#description-content-type-optional"><span class="std std-ref">Description-Content-Type</span></a></p></li>
</ul>
</div>
<p>For example, to set these values in a package’s <code class="file docutils literal notranslate"><span class="pre">setup.py</span></code> file,
use <code class="docutils literal notranslate"><span class="pre">setup()</span></code>’s <code class="docutils literal notranslate"><span class="pre">long_description</span></code> and <code class="docutils literal notranslate"><span class="pre">long_description_content_type</span></code>.</p>
<p>Set the value of <code class="docutils literal notranslate"><span class="pre">long_description</span></code> to the contents (not the path) of the README file itself.
Set the <code class="docutils literal notranslate"><span class="pre">long_description_content_type</span></code> to an accepted <code class="docutils literal notranslate"><span class="pre">Content-Type</span></code>-style value for your README file’s markup,
such as <code class="docutils literal notranslate"><span class="pre">text/plain</span></code>, <code class="docutils literal notranslate"><span class="pre">text/x-rst</span></code> (for reStructuredText), or <code class="docutils literal notranslate"><span class="pre">text/markdown</span></code>.</p>

<section id="validating-restructuredtext-markup" class="">
<h2>Validating reStructuredText markup<a class="headerlink" href="#validating-restructuredtext-markup" title="Link to this heading">¶</a></h2>
<p>If your README is written in reStructuredText, any invalid markup will prevent
it from rendering, causing PyPI to instead just show the README’s raw source.</p>
<p>Note that Sphinx extensions used in docstrings, such as
<a class="reference external" href="https://www.sphinx-doc.org/en/master/usage/restructuredtext/directives.html" title="(in Sphinx v8.3.0)"><span class="xref std std-doc">directives</span></a> and <a class="reference external" href="https://www.sphinx-doc.org/en/master/usage/restructuredtext/roles.html" title="(in Sphinx v8.3.0)"><span class="xref std std-doc">roles</span></a>
(e.g., “<code class="docutils literal notranslate"><span class="pre">:py:func:`getattr`</span></code>” or “<code class="docutils literal notranslate"><span class="pre">:ref:`my-reference-label`</span></code>”), are not allowed here and will result in error
messages like “<code class="docutils literal notranslate"><span class="pre">Error:</span> <span class="pre">Unknown</span> <span class="pre">interpreted</span> <span class="pre">text</span> <span class="pre">role</span> <span class="pre">"py:func".</span></code>”.</p>
<p>You can check your README for markup errors before uploading as follows:</p>
<ol class="arabic">
<li><p>Install the latest version of <a class="reference external" href="https://github.com/pypa/twine">twine</a>;
version 1.12.0 or higher is required:</p>
